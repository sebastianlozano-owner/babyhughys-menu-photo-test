# Baby Hughy's — menu photo test stimulus

A realistic Owner-style online-ordering menu for **Baby Hughy's Restaurant**, used as a stimulus for a
UserTesting study on how diners interpret **AI-enhanced ("replated") menu photos** at a low-price restaurant.

The menu (categories, item names, descriptions, prices) is the real live storefront. Each item can show one
of two photos:

- **`before`** — the photo currently on the menu (what diners see today)
- **`after`** — the AI-replated photo from the `replatedMenuPhoto0526` experiment

## Variant URLs

| URL | What it shows |
|---|---|
| `index.html?v=before` | The menu with **current** photos. Unlabeled — looks like a normal menu. |
| `index.html?v=after` | The same menu with **AI-enhanced** photos. Unlabeled. |
| `index.html?v=compare` | Per dish, both photos side by side as neutral **Option 1 / Option 2** (randomization handled by the test tool). |

`before` and `after` are visually identical except the photos and carry **no labels**, so a participant cannot
tell which version they're viewing — required for an unbiased monadic (between-subjects) test.

## Data
`menu.json` is generated from the live storefront page (`babyhughys.com/menu/babyhughys/`): real categories,
item names, descriptions, prices, the current image (`before`), and `imageExperimentUrls.replatedMenuPhoto0526`
(`after`). 65 items have a before+after pair. Photos are served from public CDNs.

Generated by the `usertesting` skill's photo-experiment flow.
